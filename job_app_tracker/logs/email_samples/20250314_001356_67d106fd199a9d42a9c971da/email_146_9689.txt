The endpoint allows clients to submit a scan for server-side planning, optimizing data access patterns and improving query performance. This blog explores: How to submit a scan request for planning Different scan types: Point-in-time and Incremental Handling scan responses, including completed, submitted, and failed statuses Understanding Table Scan Planning Scan planning in Iceberg allows clients to define which snapshots of a table should be read, either: Point-in-time scans → Fetch data from a specific snapshot. Incremental scans → Retrieve changes between two snapshots. This approach ensures query efficiency by minimizing unnecessary reads and improving query execution performance across large datasets.
